운영 서버 구축 방식의 종류

1. On-premise, 온 프레미스
    - 전통적인 방식
    - 내가 직접 사무실 대여 + 서버 컴퓨터 구매 + OS 구매 
    + 네트워크 환경 직접 구축 + OS 설치 + 운영 SW 설치 + 쇼핑몰 배포 + 운영
    - 자재 + 집 짓기 + 인테리어 + 가구 = 직접하는 방식
    - 장점
        - 자유롭다.
        - 보안 탁월
    - 단점
        - 전문 지식 필요 > 인력 고용
        - 운영/유지비 고가
        - 초기 구축 비용 고가

2. Iaas, Infrastructures as a Service
    - 인프라를 빌려주는 서비스
    - 서버, 스토리지, 네트워크 > H/W만 대여주는 서비스
        - 물리적인 H/W > 가상화된 H/W 제공
    - 전세, 월세
    - 장점
        - 초기 비용 약간 저렴
    - 단점
        - OS 직접 설치 > 전문 인력
        - 실행 환경 직접 구축 > 전문 인력

3. PaaS, Platform as a Service
    - 인프라 + 플랫폼 대여
    - 플랫폼 = 운영체제, 데이터베이스, 개발 도구 등..
             = 개발에 필요한 모든 환경이 이미 세팅되어 있는 서비스
    - 개발자는 직접 서버를 관리 필요X > 개발에만 집중 + 배포

4. SaaS, Software as a Service
    - 인프라 + 플랫폼 + 소프트웨어(쇼핑몰)
    - 개발자 필요X

AWS 인프라 구성 요소

1. 리전, Region
    - AWS 데이터 센터가 있는 물리적인 지역
    - ex) 서울 리전, 도쿄 리전 등..

2. 가용 영역, Availability Zone, AZ
    - 하나 리전내에 있는 독립적인 데이터 센터 그룹

3. 엣지 로케이션, Edge Location
    - 전 세계 분산된 캐시 서버
    - 데이터(이미지, 영상, 정적 파일) 콘텐츠를 사용자에게 빠르게 제공하는 서버



AWS 대표 서비스

1. Amazon EC2, Elastic Compute Cloud
- 가상 서버 서비스(VM)
- 컴퓨터 대여 서비스
- Iaas
- 종량제 > 과금(사용량)

2. Amazon RDS, Relational Database Service
- PaaS
- 관계형 데이터베이스 서비스
- 거의 대부분 DB 제공
- DB 라이센스 고가 > MySQL(저렴,무료), Oracle(고가)

3. Amazon S3, Simple Storage Service
- 객체 단위의 무제한 용량 파일 저장 스토리지
- 객체 1개의 5TB
- 99.999999999% 안전
- 용량은 무제한 > 사용한 만큼 지불(GB당 과금)
    - 업로드 트래픽(요금 거의 없음)
    - 저장된 공간(일정 요금 발생)
    - 다운로드 트래픽(일정 요금 발생)

4. Amazon EBS, Elastic Block Storage
- 스토리지 서비스
- EC2가 RDB에서 사용하는 저장 장치

        $ sudo mount /dev/nvme1n1 /data # EBS 볼륨 + 폴더 마운트

    b. EBS 볼륨을 EC2와 분리하기
        $ sudo umount /data # 우분투에서 볼륨 분리
